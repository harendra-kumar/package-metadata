Standard package metadata format
--------------------------------

The topic of `uniform or standard package file format` was discussed on
haskell-cafe mailing list. This repo contains the major points of discussion
especially the points made for and against ``.cabal``, ``.yaml`` and ``.hs`` formats.

Organized summary:

* `Motivation
  <http://www.github.com/harendra-kumar/package-metadata/uniform.rst>`_
* `Format Alternatives <http://www.github.com/harendra-kumar/package-metadata/syntax.rst>`_

* `standard package format <https://mail.haskell.org/pipermail/haskell-cafe/2016-September/124912.html>`_
